# Plan 13: Container — Containerfile, dev.sh, supporting files

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create the container environment for Podman + Wolfi including multi-stage Containerfile, dev.sh lifecycle script, LaunchAgent for macOS Podman Machine, dev.env.example, .dockerignore, and test-tool-installs.sh. Add install script modifications for linking dev.sh and LaunchAgent. Create smoke tests and wire into CI.

**Architecture:** Seven files in `container/`, modifications to both install scripts (dev.sh link + LaunchAgent setup on macOS), a smoke test in `scripts/`, and CI workflow update. The Containerfile is multi-stage (base + full). The dev.sh script handles build, shell, stop, rebuild, status, prune, clean-sessions, and macOS Podman Machine lifecycle (init-machine, machine-start/stop/status).

**Tech Stack:** Containerfile (Podman/OCI), Bash (dev.sh), XML (LaunchAgent plist)

**Design references:**
- `docs/design/container.md` — full container specification
- `docs/design/security.md` — container hardening, credential mounts, SSH agent

**Design amendments (supersede container.md for this plan):**
- **Package manager first for mise/uv:** Use Wolfi apk packages if available; fall back to checksum-pinned curl installers per security.md supply-chain requirements.
- **Missing linting tools added to Containerfile spec:** golangci-lint, gofumpt, shellcheck, shfmt, tflint, actionlint, zizmor, codeql — were in container.md but omitted from initial plan Task 1.
- **SSH_AUTH_SOCK failure mode specified:** Warn and skip mount (not exit 1) when socket is absent.
- **dev.env added to .gitignore:** Ensures secrets in dev.env cannot be committed.
- **restore() includes dev symlink cleanup:** Removes `~/.local/bin/dev` alongside LaunchAgent files.
- **clean-sessions prints risk warning:** Documents that session history may contain sensitive data.
- **install-macos.sh uses plain vars, not `local`:** Top-level code cannot use `local` keyword.

**Out of scope:**
- .devcontainer/devcontainer.json (template documented in editor-vscode.md, per-project)
- Running `podman build` or `dev init-machine` (runtime operations, not config)
- Credential files themselves (per-machine, not committed)

- [ ] **Step 3: Write podman-machine-start.sh**

Write exactly this content (with @HOMEBREW_PREFIX@ marker):

```bash
#!/bin/bash
# container/podman-machine-start.sh
# Wrapper for LaunchAgent — safe on first run, idempotent, no-loop on failure modes
set -e

# Platform guard (plist is only installed on macOS, but defence in depth)
[[ "$(uname)" == "Darwin" ]] || exit 0

# If podman isn't on PATH, exit cleanly rather than letting launchd loop
PODMAN="@HOMEBREW_PREFIX@/bin/podman"
[[ -x "$PODMAN" ]] || exit 0

# If the dotfiles machine doesn't exist, exit cleanly (user hasn't run init-machine yet)
"$PODMAN" machine list -q 2>/dev/null | grep -qx 'dotfiles' || exit 0

# Check current state; only start if stopped/configured
state=$("$PODMAN" machine inspect dotfiles --format '{{.State}}' 2>/dev/null || echo missing)
case "$state" in
  running|starting) exit 0 ;;
  stopped|configured) exec "$PODMAN" machine start dotfiles ;;
  *) exit 0 ;;
esac
```

Make executable: `chmod +x macos-dev/container/podman-machine-start.sh`

check() { if command -v "$1" >/dev/null 2>&1; then echo "  ✓ $1"; ((pass++)); else echo "  ✗ $1"; ((fail++)); fi; }

echo ""

# Base tools

            shellcheck shfmt golangci-lint gofumpt actionlint tflint zizmor; do

if [[ "${1:-}" == "--full" ]]; then
  echo ""
  echo "Full tools:"
  for tool in tmux starship lazygit btop lnav fzf zoxide bat delta fd rg nvim jq yq; do
    check "$tool"
  done
fi

echo ""
echo "Results: $pass passed, $fail failed"
exit "$( (( fail > 0 )) && echo 1 || echo 0 )"

## Task 4: Add install script modifications

**Files:**
- Modify: `macos-dev/install-macos.sh`

- [ ] **Step 1: Modify install-macos.sh**

Find the last vscode link() call and add after it:

```bash

# container dev script (Plan 13)
mkdir -p "$HOME/.local/bin"
link container/dev.sh  .local/bin/dev

# Podman Machine LaunchAgent (macOS only)
if [[ "$(uname)" == "Darwin" ]]; then
  # Substitute markers in LaunchAgent wrapper
  local wrapper_src="$DOTFILES/container/podman-machine-start.sh"
  local wrapper_dst="$HOME/.local/bin/podman-machine-start"
  sed "s|@HOMEBREW_PREFIX@|$HOMEBREW_PREFIX|g" "$wrapper_src" > "$wrapper_dst"
  chmod +x "$wrapper_dst"
  printf "  linked    %s\n" "$wrapper_dst"

  # Substitute markers in plist and install
  mkdir -p "$HOME/Library/LaunchAgents"
  local plist_src="$DOTFILES/container/io.podman.machine.plist"
  local plist_dst="$HOME/Library/LaunchAgents/io.podman.machine.plist"
  sed -e "s|@HOME@|$HOME|g" -e "s|@SCRIPT_PATH@|$wrapper_dst|g" "$plist_src" > "$plist_dst"
  printf "  linked    %s\n" "$plist_dst"

  # Load LaunchAgent (idempotent)
  launchctl bootout "gui/$(id -u)/io.podman.machine" 2>/dev/null || true
  launchctl bootstrap "gui/$(id -u)" "$plist_dst"
  log "Podman Machine LaunchAgent loaded"
fi
```

IMPORTANT: install-macos.sh uses top-level code (NOT inside a function). Do NOT use `local` keyword — use plain variable assignments. The `bash -n` check will NOT catch `local` errors at parse time; they are runtime errors.
