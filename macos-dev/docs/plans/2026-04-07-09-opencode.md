# Plan 9: OpenCode configuration — opencode.jsonc, tui.jsonc, instruction files

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create the OpenCode AI assistant configuration files (main config, TUI keybindings, and two instruction files), add symlink calls to both install scripts, and create a per-plan smoke test wired into CI. After this plan, a fresh machine will have OpenCode configured with GitHub Copilot as provider, scoped permissions, defensive bash denials, and vi-friendly TUI keybindings.

**Architecture:** Four config files in `opencode/` (including `instructions/` subdirectory), modifications to both install scripts (adding 4 `link()` calls each), a new smoke test in `scripts/`, and a CI workflow update.

**Tech Stack:** JSONC (opencode.jsonc, tui.jsonc), Markdown (instruction files)

**Design references:**
- `docs/design/opencode.md` — full OpenCode configuration spec
- `docs/design/security.md` — permission scoping, credential handling, denial rules

**Design amendments (supersede opencode.md for this plan):**
- **`session_export` binding changed:** `<leader>x` → `<leader>X` (uppercase) to prevent accidental export when using ctrl+x as Readline cut prefix followed by accidental `x`.
- **Arrow key bindings leader-prefixed:** `right`/`left`/`up` → `<leader>right`/`<leader>left`/`<leader>up` to avoid conflicting with cursor movement in the TUI input area.
- **Scoped `rm -rf` exception for scratch cleanup:** Added `"rm -rf /tmp/opencode-*": "allow"` to bash permissions so the agent can self-clean scratch files as instructed by `scratch-dirs.md` without requiring user confirmation each time.

**Out of scope for this plan:**
- critique functions (`cr`, `crw`, `crs`) — already in `.bash_aliases` (Plan 2)
- bun installation — already in Brewfile (Plan 1)
- `bun install -g opencode-ai critique` — runtime install, not a config file

**Prerequisites:**
- Plans 1–8 merged to `tool-verification-script`
- Both install scripts have `link()` helper and existing link() calls from Plans 2–8

**Branch strategy:** New worktree on branch `plan9-opencode` off `tool-verification-script`. Push to remote, merge when CI is green.
