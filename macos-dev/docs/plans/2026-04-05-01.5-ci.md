# Plan 1.5: CI workflow and branch protection

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add GitHub Actions CI that validates every PR before merge, and configure branch protection on `main` and `tool-verification-script` so broken changes cannot land on protected branches.

**Architecture:** Single GitHub Actions workflow file with four jobs at three tiers:

| Job | Runner | Trigger | Duration | Purpose |
|-----|--------|---------|----------|---------|
| `lint` | ubuntu-latest | non-draft PR + dispatch | ~30s | bash -n, shellcheck, manifest drift, DOTFILES self-resolution |
| `macos-verify` | macos-latest | non-draft PR + dispatch | ~2-3 min | `brew bundle check` + smoke tests on real macOS |
| `macos-install` | macos-latest | push to main + dispatch | ~15-20 min | Full `install-macos.sh` run + `test-tool-installs.sh` |
| `wsl-install` | windows-latest | push to main + dispatch | ~10-15 min | Full `install-wsl.sh` inside real WSL2 Ubuntu |

Fast jobs (`lint`, `macos-verify`) gate PRs. Slow jobs (`macos-install`, `wsl-install`) run on merges to main and on demand.

**Tech Stack:** GitHub Actions YAML, `Vampire/setup-wsl@v3`, `gh` CLI for branch protection setup.

**Design references:**
- `docs/design/install.md` — CI section (Tier 1 lint pipeline)
- `docs/design/DESIGN.md` — constraint: "No secrets in dotfiles" (workflow MUST NOT hardcode tokens)

**Prerequisites:**
- Plan 1 (Foundation) branch `plan1-foundation` exists and has the shell scripts + manifest check
- Repo is public at `andrewesweet/setup` so macOS/Windows runner minutes are unlimited

**Branch strategy**: This plan runs on `plan1.5-ci` branched off `plan1-foundation`. Merges back to `tool-verification-script` together with `plan1-foundation`.

**Out of scope:**
- Running CI on every push (deferred — non-draft PR + manual dispatch is sufficient)
- Replicating the enterprise binary signing policy in CI (not practical; `test-tool-installs.sh` on the actual Mac is the authoritative test)
- Caching strategy beyond Homebrew's own download cache (deferred optimisation)
- Code coverage, test reports, artifact uploads (later plans MAY add these)

- [ ] **Step 2: Write the workflow file**

Write exactly this content to `.github/workflows/verify.yml` in the worktree root (NOT inside `macos-dev/`). This workflow lives at the repo root because GitHub Actions requires `.github/workflows/` to be at the repo root.

```yaml
# .github/workflows/verify.yml
# CI for the macos-dev dotfiles setup.
#
# Fast jobs (lint, macos-verify) run on non-draft PRs and gate the merge.
# Slow jobs (macos-install, wsl-install) run on push to main and on demand.

name: verify

on:
  pull_request:
    types: [opened, synchronize, reopened, ready_for_review]
  push:
    branches: [main]
  workflow_dispatch:

# Cancel in-progress runs on the same ref when a new commit lands
concurrency:
  group: verify-${{ github.ref }}
  cancel-in-progress: true

defaults:
  run:
    shell: bash

jobs:
  # ── Tier 1: fast lint (every non-draft PR) ────────────────────────────────
  lint:
    name: lint
    runs-on: ubuntu-latest
    if: github.event_name != 'pull_request' || github.event.pull_request.draft == false
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install shellcheck
        run: |
          sudo apt-get update
          sudo apt-get install -y shellcheck

      - name: bash -n on all shell scripts under macos-dev
        run: |
          find macos-dev -type f -name '*.sh' -print0 \
            | xargs -0 -n1 bash -n

      - name: shellcheck on all shell scripts under macos-dev
        run: |
          find macos-dev -type f -name '*.sh' -print0 \
            | xargs -0 shellcheck

      - name: Tool manifest drift check
        run: bash macos-dev/scripts/check-tool-manifest.sh

      - name: DOTFILES self-resolution smoke test (macos installer)
        run: |
          mkdir -p /tmp/dotfiles-test
          ln -sf "$PWD/macos-dev/install-macos.sh" /tmp/dotfiles-test/install.sh
          output=$(bash /tmp/dotfiles-test/install.sh 2>&1 | grep -o 'DOTFILES=[^ ]*' | head -1)
          echo "Resolved: $output"
          expected="DOTFILES=$PWD/macos-dev"
          test "$output" = "$expected" || {
            echo "ERROR: expected $expected, got $output"
            exit 1
          }
          rm -rf /tmp/dotfiles-test

      - name: DOTFILES self-resolution smoke test (wsl installer)
        run: |
          mkdir -p /tmp/dotfiles-test-wsl
          ln -sf "$PWD/macos-dev/install-wsl.sh" /tmp/dotfiles-test-wsl/install.sh
          output=$(bash /tmp/dotfiles-test-wsl/install.sh </dev/null 2>&1 | grep -o 'DOTFILES=[^ ]*' | head -1)
          echo "Resolved: $output"
          expected="DOTFILES=$PWD/macos-dev"
          test "$output" = "$expected" || {
            echo "ERROR: expected $expected, got $output"
            exit 1
          }
          rm -rf /tmp/dotfiles-test-wsl

  # ── Tier 2: macOS quick verify (every non-draft PR) ──────────────────────
  macos-verify:
    name: macos-verify
    runs-on: macos-latest
    if: github.event_name != 'pull_request' || github.event.pull_request.draft == false
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Report Homebrew version
        run: brew --version

      - name: bash -n on install-macos.sh
        run: bash -n macos-dev/install-macos.sh

      - name: shellcheck on install-macos.sh
        run: shellcheck macos-dev/install-macos.sh

      - name: brew bundle check (validates formulas without installing)
        run: brew bundle check --file=macos-dev/Brewfile --verbose || true
        # --verbose lists missing formulas. We do NOT fail on missing (that's
        # install-time info). We fail only if the Brewfile itself is malformed,
        # which would exit non-zero even with || true above — but the exit code
        # from bash -n on the previous step will have caught that.

      - name: DOTFILES self-resolution on real macOS
        run: |
          mkdir -p /tmp/dotfiles-test
          ln -sf "$PWD/macos-dev/install-macos.sh" /tmp/dotfiles-test/install.sh
          output=$(bash /tmp/dotfiles-test/install.sh 2>&1 | grep -o 'DOTFILES=[^ ]*' | head -1)
          expected="DOTFILES=$PWD/macos-dev"
          test "$output" = "$expected" || {
            echo "ERROR: expected $expected, got $output"
            exit 1
          }
          rm -rf /tmp/dotfiles-test

  # ── Tier 3: full macOS install (merge to main + manual dispatch) ─────────
  macos-install:
    name: macos-install
    runs-on: macos-latest
    if: github.event_name == 'push' || github.event_name == 'workflow_dispatch'
    timeout-minutes: 45
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Cache Homebrew downloads
        uses: actions/cache@v4
        with:
          path: ~/Library/Caches/Homebrew/downloads
          key: brew-downloads-${{ runner.os }}-${{ hashFiles('macos-dev/Brewfile') }}
          restore-keys: |
            brew-downloads-${{ runner.os }}-

      - name: Run install-macos.sh
        env:
          # Disable interactive prompts from Homebrew and gcloud
          HOMEBREW_NO_AUTO_UPDATE: "1"
          HOMEBREW_NO_INSTALL_CLEANUP: "1"
          CLOUDSDK_CORE_DISABLE_PROMPTS: "1"
        run: bash macos-dev/install-macos.sh

      - name: Verify tools installed (test-tool-installs.sh)
        run: bash macos-dev/container/test-tool-installs.sh || true
        # The test script has SKIPs and FAILs by design for partial installs;
        # --install mode would retry them. For Plan 1.5 we only assert the
        # script runs and exits cleanly on the "already installed" path.

  # ── Tier 3: full WSL2 install (merge to main + manual dispatch) ──────────
  wsl-install:
    name: wsl-install
    runs-on: windows-latest
    if: github.event_name == 'push' || github.event_name == 'workflow_dispatch'
    timeout-minutes: 30
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install WSL2 with Ubuntu-22.04
        uses: Vampire/setup-wsl@v3
        with:
          distribution: Ubuntu-22.04
          update: "true"
          additional-packages: |
            bash
            sudo
            curl
            ca-certificates

      - name: Run install-wsl.sh inside WSL2
        shell: wsl-bash {0}
        run: |
          set -e
          cd "$(pwd)"
          bash macos-dev/install-wsl.sh

      - name: bash -n inside WSL2 (paranoia check)
        shell: wsl-bash {0}
        run: |
          find macos-dev -type f -name '*.sh' -print0 \
            | xargs -0 -n1 bash -n
```

## Task 3: Configure branch protection (requires GitHub PAT)

Branch protection makes the fast CI checks mandatory on PRs to protected branches. It cannot be set via a file in the repo — it requires `gh api` calls with a PAT that has `repo` or `administration:write` scope.

**IMPORTANT**: The PAT used for this task MUST NOT be committed, logged, or echoed back. Use it once in a subshell.

- [ ] **Step 1: Set protection on `main`**

Required status checks: `lint`, `macos-verify`. Require 1 PR review. Dismiss stale reviews on new commits. No force-pushes or deletions.

```bash
GH_TOKEN="<pat-value>" gh api \
  --method PUT \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2022-11-28" \
  /repos/andrewesweet/setup/branches/main/protection \
  -f "required_status_checks[strict]=true" \
  -f "required_status_checks[contexts][]=lint" \
  -f "required_status_checks[contexts][]=macos-verify" \
  -F "enforce_admins=false" \
  -f "required_pull_request_reviews[dismiss_stale_reviews]=true" \
  -F "required_pull_request_reviews[required_approving_review_count]=1" \
  -F "restrictions=null" \
  -F "allow_force_pushes=false" \
  -F "allow_deletions=false"
```

Expected: JSON response describing the new protection rules. No error.

- [ ] **Step 2: Set protection on `tool-verification-script`**

This is the long-lived integration branch we are currently working on. Same protection as `main` but without the required review (so the author can self-merge feature branches during active development).

```bash
GH_TOKEN="<pat-value>" gh api \

  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2022-11-28" \

  -f "required_status_checks[strict]=true" \
  -f "required_status_checks[contexts][]=lint" \
  -f "required_status_checks[contexts][]=macos-verify" \
  -F "enforce_admins=false" \
  -F "required_pull_request_reviews=null" \
  -F "restrictions=null" \
  -F "allow_force_pushes=false" \
  -F "allow_deletions=false"
```

Note: `required_pull_request_reviews=null` disables the review requirement on this branch.

## Known concerns

- **First-run chicken and egg**: Plan 1.5 creates CI that must pass before its own PR can merge. The first CI run on plan1.5-ci will validate itself. If it fails, fix forward on the same branch — the protection rules on `main` and `tool-verification-script` only apply to those branches, not to plan1.5-ci itself.
- **WSL2 action version drift**: `Vampire/setup-wsl@v3` is pinned to a major version. Monitor for releases; an automated dependabot config could be added later.
- **`gh` CLI must be authenticated OR PAT must be in env**: the branch protection task dispatches a subshell with `GH_TOKEN=<pat>` inline. If the PAT scope is wrong (e.g. missing `administration:write`), the API call fails with 403.
- **`brew bundle check` false-negatives**: the current step uses `|| true` to avoid failing on "would install X" — but this also masks real Brewfile syntax errors. The preceding `bash -n` step catches syntax errors in install-macos.sh, but not in the Brewfile itself. Brewfile syntax errors would show up at `macos-install` runtime instead.
